Multi-Paradigm Problem Solving

DOCTRINE

Agent capability for applying multiple problem-solving paradigms including functional, logical, heuristic, epistemic, and formal approaches to complex challenges

Overview

Multi-Paradigm Problem Solving is a foundational capability of the Prismatic Platform that enables agents to select, combine, and dynamically switch between multiple problem-solving paradigms when tackling complex challenges. Rather than forcing every problem through a single reasoning framework, the platform recognizes that different problem domains demand different cognitive tools — and that the most robust solutions often emerge from the intersection of multiple approaches.

This capability draws on decades of research in artificial intelligence, programming language theory, and cognitive science. The key insight is that no single paradigm dominates across all problem types: functional decomposition excels at data transformation pipelines, logical inference handles constraint satisfaction and knowledge representation, heuristic search navigates vast combinatorial spaces, formal verification ensures mathematical correctness, and epistemic reasoning manages uncertainty and incomplete information.

Within Prismatic, every agent from Level 2 upward has access to the multi-paradigm selection protocol, while Level 3+ agents can dynamically compose paradigms within a single problem-solving session. The platform's NABLA axioms ensure that paradigm selection itself is subject to epistemic rigor — agents must justify why a particular paradigm was chosen and demonstrate that the selection process preserves signal plurality.

Core Paradigms

The platform supports five primary problem-solving paradigms, each with distinct strengths, characteristic problem domains, and integration points within the agent hierarchy.

Functional Paradigm

The functional paradigm treats computation as the evaluation of mathematical functions, emphasizing immutability, pure transformations, and compositional reasoning. In Prismatic, this paradigm is deeply embedded in the Elixir/OTP foundation.

PropertyDescription
Core PrinciplePure functions with no side effects
Data ModelImmutable values, persistent data structures
CompositionFunction composition, pipeline operators
Error Handling{:ok, value} / {:error, reason} tagged tuples
ConcurrencyMessage-passing between isolated processes
Ideal ForData transformation, ETL pipelines, stream processing

Agents apply the functional paradigm when problems can be decomposed into a sequence of pure transformations. The Elixir pipe operator (|>) serves as both a programming construct and a cognitive model — agents reason about solutions as data flowing through transformation stages.

# Functional paradigm in action: claim processing pipeline
artifact
|> Ingest.Parser.extract_text()
|> Extract.ClaimExtractor.identify_claims()
|> Enum.map(&Nabla.Evaluator.score/1)
|> Enum.filter(&above_threshold?/1)
|> BlackboardBridge.store_batch()

Logical Paradigm

The logical paradigm uses formal rules, constraint satisfaction, and deductive inference to solve problems. It excels when the problem domain can be expressed as a set of logical statements with clear inference rules.

PropertyDescription
Core PrincipleDeductive inference from axioms and rules
Data ModelFacts, rules, and queries
CompositionRule chaining, backward/forward chaining
Error HandlingFailure as information (closed-world assumption)
ConcurrencyParallel rule evaluation
Ideal ForCompliance checking, constraint satisfaction, knowledge representation

Within Prismatic, the 3NL (Three-Nesting-Level) pipeline is the primary expression of logical reasoning. Agents operating in the logical paradigm construct rule sets that encode domain knowledge, then query these rules to derive conclusions. The Trinity Gate verification system itself is an application of logical paradigm thinking — it uses deductive rules to verify structural, logical, and formal consistency.

Heuristic Paradigm

The heuristic paradigm applies pattern-based optimization, evolutionary search, and approximate reasoning when exact solutions are computationally intractable or when the problem space is too large for exhaustive search.

PropertyDescription
Core PrincipleGood-enough solutions via intelligent search
Data ModelSolution candidates, fitness landscapes
CompositionPopulation-based iteration, tournament selection
Error HandlingGraceful degradation, anytime algorithms
ConcurrencyParallel population evaluation
Ideal ForOptimization, scheduling, resource allocation, evolutionary design

The platform's self-evolution system (prismatic_evolve) is the canonical example of heuristic reasoning. The fitness evaluator maintains a multi-dimensional fitness vector — deliberately avoiding scalar reduction to prevent Goodhart's Law attacks — while the evolutionary selector uses tournament selection with hard constraints to explore the solution space. Anti-singularity safeguards ensure that heuristic optimization never collapses the fitness landscape into a single-dimensional race.

Formal Paradigm

The formal paradigm uses mathematical proof, type theory, and property-based testing to establish guarantees about system behavior. It provides the highest level of assurance but requires the most precise problem specification.

PropertyDescription
Core PrincipleMathematical proof of correctness
Data ModelPropositions, theorems, type inhabitants
CompositionProof composition, lemma reuse
Error HandlingProof failure reveals specification gaps
ConcurrencyIndependent proof obligations
Ideal ForSafety-critical systems, protocol verification, invariant enforcement

Prismatic integrates formal verification through Lean4 proofs in the QEVE (Quantitative Epistemic Verification Engine) system and through property-based testing using StreamData. The White Team agents specialize in formal paradigm reasoning, constructing progressive proofs from L0 (basic assertions) through L5 (full formal verification). The Trinity Gate's Gate 3 (Formal Necessity) is the system's commitment to formal reasoning as a non-optional verification layer.

Epistemic Paradigm

The epistemic paradigm applies uncertainty reasoning, evidence weighting, and belief management to problems where information is incomplete, contradictory, or evolving. This paradigm is unique to Prismatic and emerges from the NABLA infinity framework.

PropertyDescription
Core PrincipleUncertainty is data, not noise
Data ModelBeliefs with confidence intervals, evidence graphs
CompositionEvidence accumulation, Bayesian updating
Error Handling"I don't know" is a valid state
ConcurrencyIndependent evidence evaluation, society-based plurality
Ideal ForIntelligence analysis, risk assessment, decision support under uncertainty

The epistemic paradigm is governed by the seven NABLA Axioms: signal plurality, contradiction preservation, absence as information, time decay, unknown validity, source independence, and provenance mandate. Agents operating in the epistemic paradigm must demonstrate compliance with all seven axioms, and their conclusions must pass the Trinity Gate before being accepted into the platform's knowledge base.

Paradigm Selection Protocol

The selection of which paradigm(s) to apply for a given problem is itself a structured decision process, not an ad-hoc choice. The platform implements a five-stage selection protocol.

Stage 1: Problem Classification

The incoming problem is classified along multiple dimensions:

  • Domain type: Data transformation, constraint satisfaction, optimization, verification, or uncertainty management
  • Completeness: Is the problem fully specified, partially specified, or open-ended?
  • Tractability: Can an exact solution be computed in polynomial time?
  • Safety criticality: What are the consequences of an incorrect solution?
  • Evidence availability: How much relevant data exists?

Stage 2: Complexity Assessment

The problem's computational complexity and structural properties are measured:

  • State space size: How large is the solution space?
  • Constraint density: How constrained is the feasible region?
  • Decomposability: Can the problem be broken into independent subproblems?
  • Temporal dynamics: Is the problem static or evolving?

Stage 3: Paradigm Matching

Based on classification and complexity, candidate paradigms are ranked:

Problem Input --> Classify Domain --> Measure Complexity --> Rank Paradigms
     |                  |                    |                    |
  Categorize       Assess scale         Evaluate            Score each
  by type         and structure        tractability         paradigm fit
Problem ProfilePrimary ParadigmSecondary Paradigm

Constraint satisfactionLogicalHeuristic

Multi-objective under uncertaintyEpistemic + HeuristicFormal

Stage 4: Composition Strategy

When multiple paradigms are selected, the composition strategy determines how they interact:

  • Sequential: Apply paradigms in order (e.g., functional preprocessing then logical inference then formal verification)
  • Parallel: Apply paradigms independently and merge results (e.g., heuristic search alongside logical constraint solving)
  • Nested: Use one paradigm within another (e.g., epistemic uncertainty wrapping formal proofs)
  • Adaptive: Switch paradigms based on intermediate results

Stage 5: Validation

The selected paradigm composition is validated against the platform's quality requirements:

  • Does the selection preserve NABLA axiom compliance?
  • Does the composition strategy support Trinity Gate verification?
  • Are the NO MERCY quality standards achievable with the selected paradigms?
  • Is the selection evidence-based per NO DOUBTS requirements?

Society-Based Paradigm Diversity

The four epistemic societies in prismatic_epistemic_world each emphasize different paradigm preferences, ensuring that multi-paradigm diversity is maintained at the organizational level:

  • Skeptics: Favor logical and formal paradigms (falsification-focused)
  • Archivists: Favor epistemic and functional paradigms (historical context)
  • Counterfactualists: Favor heuristic and epistemic paradigms (alternative generation)
  • Red Team: Favor adversarial heuristic paradigms (attack simulation)

This society structure is an anti-singularity safeguard — it prevents the platform from converging on a single "winning" paradigm and losing the cognitive diversity that makes multi-paradigm solving effective.

Integration Points

Multi-paradigm problem solving integrates deeply with other platform capabilities:

  • NABLA Axioms: Govern epistemic paradigm reasoning and ensure paradigm selection itself is evidence-based
  • Trinity Gate: Provides formal verification layer that validates solutions regardless of which paradigm produced them
  • NO MERCY: Enforces that paradigm selection leads to complete, production-ready solutions with zero tolerance for incomplete implementations
  • NO DOUBTS: Requires that paradigm selection decisions are fully investigated and evidence-based before execution
  • Autonomous Self-Healing: Uses heuristic paradigm for self-improvement while formal paradigm validates that evolution preserves hard constraints
  • Intelligence Synthesis: Applies epistemic paradigm to intelligence analysis with multi-source evidence fusion

Design Principles

The multi-paradigm solving capability is guided by several design principles that distinguish it from simpler "tool selection" approaches:

  1. No paradigm is universal: Every paradigm has blind spots. The platform explicitly rejects the notion that any single approach is sufficient for all problems.

  2. Paradigm selection is itself epistemic: The choice of which paradigm to apply is subject to the same rigor as any other platform decision — it must be evidence-based, traceable, and pass verification.

  3. Diversity is a hard constraint: The evolutionary system treats paradigm diversity as a hard constraint (not a soft objective), preventing convergence on a monoculture of reasoning approaches.

  4. Composition over selection: The most powerful solutions often emerge from composing multiple paradigms rather than selecting a single "best" one.

  5. Meta-cognition: Agents must be able to reason about their own reasoning process, recognizing when a paradigm is failing and switching to an alternative before committing to a suboptimal path.
