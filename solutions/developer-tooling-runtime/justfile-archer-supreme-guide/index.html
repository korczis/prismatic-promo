📋 Recipe Reference

Core Development Recipes

just setup

Purpose: Complete development environment setup

What it does:

  1. Installs runtime tools via ASDF (Erlang, Elixir, Node.js)
  2. Installs Elixir dependencies (mix deps.get)
  3. Starts required services (PostgreSQL, Redis, Meilisearch)
  4. Creates and migrates database
  5. Sets up MCP infrastructure
  6. Verifies entire setup

Prerequisites: Docker, ASDF (recommended)

Output:

  → Setting up runtime environment...
    ✅ Runtime tools installed via ASDF
  → Installing dependencies...
    ✅ Dependencies installed
  → Starting services...
    ✅ Services started and ready
  → Setting up database...
    ✅ Database ready
  → Setting up MCP infrastructure...
    ✅ MCP infrastructure ready
  → Verifying setup...
    ✅ Compilation verified
    ✅ Services verified

✅ Setup complete! Ready for development.

Next steps:
  • Run 'just serve' to start development server
  • Run 'just test' to run test suite
  • Run 'just quality' to check code quality

just compile [env]

Purpose: Smart compilation with warning management

Parameters:

  • env: Target environment (dev, test, prod) - default: dev

Features:

  • Syntax error detection: Fails fast on syntax errors
  • Warning management: Treats warnings as errors
  • Environment-specific: Compiles for target environment
  • Intelligent feedback: Suggests fixes for common issues

Examples:

just compile          # Compile for dev environment
just compile test     # Compile for test environment
just compile prod     # Compile for production

just test [type]

Purpose: Intelligent testing with coverage tracking

Parameters:

  • type: Test type (all, unit, integration, e2e, fast) - default: all

Features:

  • Selective testing: Run specific test suites
  • Coverage tracking: Automatic coverage reports
  • Fast feedback: Skip slow tests when needed
  • Environment setup: Ensures test compilation

Examples:

just test             # Run all tests with coverage
just test unit        # Run unit tests only
just test integration # Run integration tests only
just test fast        # Skip slow tests

just serve [port]

Purpose: Development server with intelligent startup

Parameters:

  • port: Server port - default: 4000

Features:

  • Service auto-start: Ensures all required services are running
  • Compilation check: Verifies code compiles before starting
  • Database setup: Automatically sets up database if needed
  • Health monitoring: Pre-flight checks for optimal startup

Examples:

just serve            # Start on port 4000
just serve 3000       # Start on port 3000
just serve-iex        # Start with IEx console

Quality & Compliance Recipes

just quality

Purpose: Unified quality gates with parallel execution

Features:

  • Parallel execution: Runs Archer scan and Credo simultaneously
  • Comprehensive checks: Compilation, quality analysis, test coverage
  • Fast feedback: Optimized for speed and efficiency
  • Pass/fail gates: Clear success/failure indicators

Process:

  1. Parallel Phase: Archer scan + Credo analysis
  2. Validation Phase: Quality gate validation
  3. Coverage Phase: Test coverage verification (85% threshold)

just fix

Purpose: Automated quality issue resolution

Features:

  • Syntax fixes: Common syntax error resolution
  • Code formatting: Automatic formatting via mix format
  • Warning fixes: Archer elimination for common warnings
  • Verification: Attempts compilation to verify fixes

Process:

  1. Fix syntax issues
  2. Apply code formatting
  3. Fix compilation warnings
  4. Verify with compilation attempt

just lint [tool]

Purpose: Comprehensive linting with multiple tools

Parameters:

  • tool: Linting tool (all, credo, archer, dialyzer) - default: all

Features:

  • Multiple linters: Credo, Archer, Dialyzer support
  • Selective linting: Run specific linters
  • Strict mode: Uses strict checking for maximum quality

Purpose: Unified service management

🎛️ Configuration

Environment Variables

The justfile respects these environment variables:

# Default environment (dev/test/prod)
export MIX_ENV=dev

# Database configuration
export POSTGRES_USER=prismatic
export POSTGRES_DB=prismatic_dev

# Service URLs
export REDIS_URL=redis://localhost:6379
export MEILISEARCH_URL=http://localhost:7700

.env File Support

Create a .env file in the project root:

# .env
MIX_ENV=dev
POSTGRES_USER=prismatic
POSTGRES_DB=prismatic_dev
REDIS_URL=redis://localhost:6379
MEILISEARCH_URL=http://localhost:7700

The justfile automatically loads these variables.

🔍 Troubleshooting

Common Issues

"Command not found: just"

Solution: Install Just command runner

# macOS
brew install just

# Ubuntu/Debian
wget -qO - 'https://proget.makedeb.org/debian-feeds/prebuilt-mpr.pub' | gpg --dearmor | sudo tee /usr/share/keyrings/prebuilt-mpr-archive-keyring.gpg 1> /dev/null
echo "deb [arch=all,amd64,i386 signed-by=/usr/share/keyrings/prebuilt-mpr-archive-keyring.gpg] https://proget.makedeb.org prebuilt-mpr $(lsb_release -cs)" | sudo tee /etc/apt/sources.list.d/prebuilt-mpr.list
sudo apt update
sudo apt install just

"Services not starting"

Solution: Check Docker and docker-compose

# Check Docker
docker --version
docker-compose --version

# Check services manually
just services status
just emergency  # For detailed diagnostics

"Compilation warnings treated as errors"

Solution: Use automated fixes

just fix          # Automated issue resolution
just fix-warnings # Specific warning fixes
just compile dev  # Retry compilation

"Database connection issues"

Solution: Reset database setup

just services restart postgres
just _setup_database
just doctor  # Verify fix
