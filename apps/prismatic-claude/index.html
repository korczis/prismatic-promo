Prismatic Claude

Claude AI integration with stack-based conversation management and session lifecycle

680 files AI

Abstract

Prismatic Claude provides the deep integration layer between the Prismatic Platform and Anthropic's Claude AI models, implementing stack-based conversation management, session lifecycle hooks, and the infrastructure for AI-assisted development, quality enforcement, and autonomous platform evolution. The StackConversation GenServer implements a frame-based conversation model where every interaction is tracked as an immutable frame on a stack, supporting operations including checkpoint creation, conversation forking, frame popping, and checkpoint restoration. The SessionLifecycle GenServer manages hooks that execute at four session boundaries (session_start, pre_command, post_command, session_end), ensuring quality gates, evolution triggers, and context preservation happen automatically. A circuit breaker pattern protects against cascading failures from flaky mix tasks, auto-opening after three consecutive failures and auto-resetting after 60 seconds. The application manages bidirectional communication between human operators and the 404+ agent ecosystem through Prismatic Agents.

1. Introduction

1.1 Problem Statement

AI-assisted development requires structured conversation management that preserves context across interactions, enforces quality gates at session boundaries, and enables exploration of alternative approaches through conversation branching. Without structured conversation tracking, context is lost between sessions, quality checks are skipped, and the relationship between AI interactions and code changes becomes untraceable.

Prismatic Claude solves this by providing an OTP-compliant infrastructure for conversation state management, session lifecycle automation, and quality DNA persistence across sessions.

1.2 Design Goals

  1. Stack-based conversation tracking -- every interaction is an immutable frame on a stack, enabling replay, branching, and rollback.
  2. Session lifecycle automation -- hooks at session boundaries ensure quality gates, evolution checks, and context preservation.
  3. Circuit breaker protection -- flaky mix tasks cannot cascade into session lifecycle failures.
  4. Quality DNA persistence -- cross-session quality state enables trend analysis and intervention tracking.
  5. Agent integration -- structured dispatch to the 404+ agent ecosystem.
  6. Telemetry coverage -- every significant operation emits telemetry events for monitoring.

1.3 Scope

Prismatic Claude covers conversation management, session lifecycle, and AI integration infrastructure. It does not implement the AI models themselves (which are external) or agent-specific logic (which resides in Prismatic Agents).

2.4 Data Flow

User input triggers frame creation in StackConversation. The frame records input summary, assumptions, and decisions. SessionLifecycle executes registered hooks for the current phase. Hooks run mix tasks (quality gates, evolution scans) in isolated processes with timeout protection. The circuit breaker prevents cascading failures. After agent dispatch and response generation, the frame is completed with output summary and persisted to ETS and disk.

3. Implementation

3.1 Key Algorithms

Frame Immutability. Once a frame is created and completed, it cannot be modified. The pop operation removes frames from the stack but does not alter them. The fork operation creates a new branch starting from a specified frame, preserving the original stack.

Circuit Breaker. The SessionLifecycle circuit breaker tracks consecutive failures per hook. After three failures, the circuit opens, bypassing the hook for 60 seconds. After the reset timeout, the circuit enters half-open state, allowing one test execution. Success closes the circuit; failure reopens it.

6. Testing Strategy

6.1 Unit Tests

StackConversation tests verify frame creation, immutability, pop, fork, checkpoint, and goto operations. Circuit breaker tests verify state transitions through closed, open, and half-open states.

6.2 Integration Tests

Full session lifecycle tests exercise hook registration, execution, and circuit breaker behavior with simulated mix task failures.

6.3 Property-Based Testing

StreamData generators produce random frame sequences to verify that stack operations maintain consistency and that checkpoint/goto operations are reversible.

7. Security Considerations

7.1 Threat Model

Session context may contain sensitive information. Disk persistence is restricted to the project directory. Frame content is not encrypted at rest.

7.2 Access Control

Prismatic Claude is a development-time tool with local filesystem access. No network-facing access control.

8. Operational Considerations

8.1 Deployment

Deploys as part of the umbrella release. Persistence directories are created on first run.

8.2 Monitoring

Telemetry events: [:prismatic_claude, :stack_conversation, :frame_created], [:prismatic_claude, :session_lifecycle, :hook_executed], [:prismatic_claude, :session_lifecycle, :circuit_opened].
