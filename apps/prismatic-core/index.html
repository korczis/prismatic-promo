Prismatic Core

Foundation module providing core abstractions, protocols, and shared utilities

180 files Foundation

Overview

Prismatic Core is the foundational module of the Prismatic Platform, defining core abstractions, shared protocols, base behaviors, and utility functions that every other application depends on. The module establishes the platform's type system through entity protocols for consistent data access, serialization protocols for storage adapter compatibility, validation protocols for data integrity, and comparison protocols for ordering.

A platform comprising 90 OTP applications requires a shared foundation of types, protocols, and utilities. Without centralization, each application would define its own entity representations, error types, and utility functions, leading to incompatible interfaces, duplicated code, and inconsistent behavior. Cross-application communication requires agreed-upon data structures and protocols. Prismatic Core provides this shared foundation, ensuring all applications speak the same language for entities, errors, configuration, and common operations.

It provides platform-wide configuration management, structured error types following the {:ok, result} | {:error, reason} convention, cryptographic utilities (hashing, HMAC, encryption), string manipulation with Czech diacritics support, date/time handling with timezone awareness, and collection utilities. Prismatic Core has zero external dependencies beyond the Elixir standard library and OTP, ensuring it can be depended upon by every application in the umbrella without introducing transitive dependency conflicts. The module also includes integrations for the blackboard pattern, Monte Carlo simulation, mycelial networking, git forensics, and feature flags -- all implemented as part of the core runtime.

Architecture

Prismatic Core is a pure library application with no processes. All functions are stateless and thread-safe. No supervision tree, no GenServers, no ETS tables.

Applications import Core modules and call functions directly. Protocol dispatch is resolved at compile time for known types and at runtime for dynamic dispatch.

Czech diacritics normalization provides bidirectional conversion between accented Czech characters and their ASCII equivalents, enabling fuzzy matching in entity resolution. All errors are represented as structs with a type atom, a human-readable message, and optional details map, enabling pattern matching on error types in consuming applications.

defprotocol PrismaticCore.Entity do
  @spec id(t()) :: String.t()
  def id(entity)

  @spec type(t()) :: atom()
  def type(entity)

  @spec metadata(t()) :: map()
  def metadata(entity)
end

defmodule PrismaticCore.Error do
  @type t :: %__MODULE__{
    type: atom(),
    message: String.t(),
    details: map()
  }
  defstruct [:type, :message, details: %{}]

  def not_found(resource, id), do: %__MODULE__{type: :not_found, message: "#{resource} #{id} not found"}
  def validation_failed(errors), do: %__MODULE__{type: :validation_failed, details: %{errors: errors}}
  def unauthorized(reason), do: %__MODULE__{type: :unauthorized, message: reason}
end

Configuration

config :prismatic_core,
  environment: :prod,
  timezone: "Europe/Prague",
  locale: :cs,
  crypto_key_env: "PRISMATIC_CRYPTO_KEY"

Configuration is minimal by design. Environment determines runtime behavior (dev/test/prod). Timezone defaults to Prague for Czech-centric operations. Locale setting influences string handling and date formatting. The crypto key is sourced from environment variables, never committed to source code.

API Reference

# Entity protocol implementation
defimpl PrismaticCore.Entity, for: MyStruct do
  def id(entity), do: entity.uuid
  def type(_entity), do: :my_entity
  def metadata(entity), do: %{created_at: entity.inserted_at}
end

# Configuration access
config = PrismaticCore.Config.get(:prismatic, :feature_flags)
PrismaticCore.Config.get(:prismatic, :max_retries, 3)

# Structured error handling
{:error, PrismaticCore.Error.not_found("User", user_id)}

# Cryptographic utilities
hash = PrismaticCore.Crypto.sha256(data)
hmac = PrismaticCore.Crypto.hmac_sha256(key, message)

# Czech string normalization
normalized = PrismaticCore.Utils.String.normalize_czech("Zakon o kyberneticke bezpecnosti")

Testing

Protocol implementation tests verify correct dispatch for all supported types. Error type tests verify pattern matching and message formatting. Crypto tests verify hash computation against known test vectors. Cross-application integration tests verify that entities serialized by one application can be deserialized by another through the shared protocols.

Property-based tests use StreamData generators to produce random strings, dates, and entity data, verifying that serialization roundtrips preserve data integrity and that Czech normalization is idempotent. Crypto utilities are tested for constant-time comparison to prevent timing attacks.

Integration Points

Prismatic Core has zero dependencies beyond Elixir and OTP. Every Prismatic application depends on Core -- it is the root of the dependency tree. As a pure library, Core has no inter-process communication. All functionality is self-contained with no external integrations.
