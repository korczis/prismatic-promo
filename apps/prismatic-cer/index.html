Prismatic CER

Compliance and Evidence Repository for regulatory audit trail management

105 files Compliance

Overview

Prismatic CER (Compliance and Evidence Repository) provides structured storage and management of compliance evidence, audit artifacts, and regulatory documentation. It supports NIS2, GDPR, and custom compliance frameworks by maintaining a tamper-proof repository of evidence collected from platform operations.

Regulatory compliance increasingly demands continuous, verifiable evidence rather than point-in-time audits. CER addresses this by automatically collecting evidence from platform operations -- security scans, configuration assessments, incident response actions, access control changes -- and mapping each piece of evidence to the specific regulatory requirements it satisfies. This transforms compliance from a periodic manual exercise into an always-current automated process.

The repository implements tamper-proof storage using content-addressable hashing. Every evidence artifact is stored with a cryptographic hash, and the hash chain creates an immutable audit trail that auditors can independently verify. Evidence progresses through a defined lifecycle from collection through review to approval, with full provenance tracking at each stage. This design ensures that organizations can demonstrate not just current compliance, but the complete history of how compliance was achieved and maintained.

Key Features

Evidence Management

  • Structured collection: Evidence is automatically classified by framework, requirement, and evidence type upon collection from platform operations
  • Tamper-proof storage: Content-addressable storage with SHA-256 hash verification ensures evidence integrity throughout its lifecycle
  • Lifecycle tracking: Evidence moves through collection, review, approval, and archival stages with timestamps and responsible parties recorded at each transition
  • Automated collection: Platform operations automatically generate evidence artifacts -- scan results become NIS2 Article 21 evidence, access logs become GDPR Article 30 records

Framework Support

  • NIS2 Directive: Complete mapping of evidence requirements for all NIS2 articles including risk management (Article 21), incident reporting (Article 23), and supply chain security (Article 22)
  • GDPR compliance: Data processing records, consent management evidence, DPIA documentation, and data subject request tracking
  • ZKB 264/2025: Czech cybersecurity requirements including asset inventory evidence, vulnerability management records, and incident response documentation
  • Custom frameworks: Extensible framework definition allowing organizations to define their own compliance requirements and evidence mappings

Audit Support

  • Full evidence chain: Auditors can trace any compliance claim back through the evidence chain to the original source data
  • Report generation: Automated compliance reports in PDF, HTML, and structured JSON formats with evidence references and gap identification
  • Gap analysis: Continuous comparison of collected evidence against framework requirements, highlighting areas where evidence is missing or stale
  • Auditor access portal: Read-only access interface for external auditors with scoped permissions and activity logging

Architecture

CER is designed as a write-heavy, read-optimized repository. Evidence collection happens continuously from multiple platform sources, while read operations are concentrated during audit periods and report generation. The architecture separates the ingestion pipeline from the query layer, allowing high-throughput evidence collection without impacting report generation performance.

Platform Operations --> Evidence Collector --> Hash Generator --> Evidence Store (Ecto)
                              |                                        |
                        Classification Engine                   Framework Mapper
                        (framework, requirement,                (NIS2, GDPR, ZKB)
                         evidence type)                               |
                                                              Report Generator
                                                              /       |       \
                                                          PDF       HTML      JSON

The classification engine uses rule-based mapping to automatically associate evidence with regulatory requirements. The framework mapper maintains the complete requirement hierarchy for each supported regulation.

Integration

CER collects evidence from across the platform. Prismatic Perimeter provides security assessment evidence, Prismatic Tracking supplies entity monitoring records, Prismatic Suppression provides alert handling audit trails, and Prismatic Storage Ecto serves as the persistence layer for the evidence repository. The Prismatic Web LiveView interface provides the auditor access portal and compliance dashboard.

Usage

# Store compliance evidence
PrismaticCer.store_evidence(%{
  framework: :nis2,
  requirement: "Article 21.2(a)",
  evidence_type: :policy_document,
  document: document_binary,
  metadata: %{collected_at: DateTime.utc_now()}
})

# Generate compliance report
{:ok, report} = PrismaticCer.compliance_report(:nis2, format: :pdf)

# Run gap analysis for a framework
{:ok, gaps} = PrismaticCer.gap_analysis(:gdpr,
  as_of: Date.utc_today())
# => %{total_requirements: 87, covered: 71, gaps: 16, stale: 4}

# Verify evidence chain integrity
{:ok, :verified} = PrismaticCer.verify_chain(
  framework: :nis2,
  requirement: "Article 21.2(a)")

# Create auditor access
{:ok, access} = PrismaticCer.grant_auditor_access(
  auditor: "auditor@firm.com",
  scope: [:nis2, :gdpr],
  expires: ~U[2026-04-01 00:00:00Z])
