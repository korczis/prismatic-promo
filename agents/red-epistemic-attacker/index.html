red-epistemic-attacker

Specialist agent simulating truth distortion and source poisoning attacks against epistemic systems. Measures the impact of factual corruption on downstream belief accuracy, confidence scores, and decision quality within strict sandbox isolation.

Executive Summary

The Red Epistemic Attacker is the Prismatic Platform's L2 tactical specialist for truth distortion and source poisoning simulation -- the two most direct attack vectors against epistemic system integrity. While the Red Commander orchestrates the full spectrum of adversarial scenarios across five attack primitives, the Epistemic Attacker focuses exclusively on corrupting the factual foundations and source reliability that the platform's belief system depends on. It answers a critical question: what happens to the platform's decision-making when the information it trusts is systematically distorted?

The agent operates within the strictest safety constraints available to any operational agent: sandbox-only execution, synthetic data exclusively, zero network access, no persistent state changes, ethics checks every 15 seconds, and read-only tool access (it cannot even write files or execute shell commands). Despite these constraints, it exercises genuine adversarial capability within its sandbox environment, measuring precisely how truth distortion and source poisoning degrade downstream belief accuracy, confidence scores, and autonomous decision quality.

The Epistemic Attacker reports exclusively to the Red Commander, producing structured AttackResult artifacts that quantify the impact of each simulation. These results feed into the Red Commander's finding emission pipeline, ultimately reaching the Purple Coordinator and Blue Commander through the Color Teams synthesis loop.

Technical Architecture

The Epistemic Attacker's architecture implements two specialized attack simulation engines with a rigorous measurement framework.

Truth Distortion Engine. This engine simulates systematic alteration of factual content within synthetic knowledge bases. Five distortion techniques are available, each targeting a different aspect of factual integrity.

TechniqueMethodMeasurement
Selective Fact ReplacementReplace specific facts with distorted versionsPer-fact accuracy delta
Noise InjectionAdd random factual noise to knowledge baseSignal-to-noise ratio degradation
Gradient DistortionProgressively degrade factual accuracyAccuracy curve over distortion steps
Contradiction InjectionInsert logically inconsistent factsConsistency check failure rate
Omission AttackSelectively remove critical factsDecision quality with missing data

Each technique is calibrated to produce measurable effects that can be compared against pre-attack baselines. The gradient distortion technique is particularly valuable for identifying detection thresholds -- the level of distortion at which the platform's quality mechanisms first detect corruption.

Source Poisoning Engine. This engine corrupts information sources at their origin, targeting the trust and provenance mechanisms that the NABLA Infinity framework relies on. Four poisoning techniques simulate different source integrity attacks.

TechniqueMethodMeasurement
Reliability Score ManipulationInflate or deflate source reliability ratingsDownstream confidence score deviation
Provenance Chain TamperingAlter or sever provenance metadataTrust chain verification failure rate
Source Authority ImpersonationPresent low-authority sources as high-authorityFalse belief acceptance rate
Cross-ContaminationPropagate poisoned data across source boundariesContamination spread radius

Source poisoning is particularly dangerous to epistemic systems because it undermines the trust infrastructure that all downstream processing depends on. If source reliability scores are inflated, every belief derived from that source carries false confidence.

Epistemic Integrity Testing Framework. Beyond individual attack techniques, the Epistemic Attacker stress-tests specific platform subsystems against corruption.

Operational Model

The Epistemic Attacker operates exclusively under Red Commander directives, following a structured execution protocol for each attack simulation.

Phase 1: Directive Reception. The commander issues an AttackDirective specifying the attack type (truth distortion or source poisoning), technique, target description, and scenario parameters. The Epistemic Attacker validates the directive against its capability set and confirms sandbox readiness.

Phase 2: Baseline Measurement. Before applying any attack, the agent measures the target's current state across four metrics: integrity (factual consistency of the knowledge base), confidence (aggregate confidence scores across beliefs), accuracy (correctness of derived conclusions), and consistency (logical consistency across the belief network). These baseline metrics provide the reference against which attack impact is measured.

Phase 3: Attack Execution. The selected technique is applied within PrismaticDark.Sandbox with :strict isolation. The attack proceeds according to the scenario parameters -- for gradient distortion, this means stepwise degradation with measurements at each step; for selective fact replacement, this means targeted alteration of specific facts with immediate effect measurement.

Phase 4: Impact Measurement. After attack execution, the same four metrics are measured again. The delta between baseline and post-attack values quantifies the attack's impact. Detection latency is also measured -- how many processing cycles elapsed before the platform's quality mechanisms detected the corruption (or :undetected if the corruption was not detected at all).

Phase 5: Result Packaging. A structured AttackResult artifact is produced containing the attack identifier, type, technique, target description, pre-attack metrics, post-attack metrics, degradation deltas, hypothesis confirmation status, detection latency, sandbox verification flag, synthetic data confirmation, and audit trail identifier. This artifact is transmitted exclusively to the Red Commander.

Authority Framework

The Red Epistemic Attacker holds L2 Tactical Specialist authority with the most restricted operational scope of any agent that executes active operations.

Execution Authority. The agent may execute truth distortion and source poisoning simulations exclusively within PrismaticDark.Sandbox, exclusively using synthetic data, and exclusively under Red Commander directive. It has no self-directed operation capability -- every execution must be authorized by a commander directive.

Measurement Authority. The agent may measure any metric within its sandbox environment. Measurements are authoritative records of attack impact that feed into the Red Commander's finding pipeline.

Reporting Authority. The agent reports exclusively to the Red Commander. No direct communication with Blue, Purple, White, or any other team is permitted. All external visibility of Epistemic Attacker results is mediated by the Red Commander's finding emission and validation pipeline.

Tool Restrictions. The agent has access only to read-only tools: Read, Grep, Glob, and Task. Write, Edit, and Bash tools are explicitly forbidden, preventing any modification of system state outside the sandbox environment. This is the most restrictive tool set of any agent that performs active operations.

Safety Architecture

The Epistemic Attacker operates under seven absolute constraints with zero tolerance for violation.

  1. Sandbox Only: All execution through PrismaticDark.Sandbox with :strict isolation level
  2. Synthetic Data Only: No real data, no PII, no production state
  3. Zero Network Access: No network connectivity of any kind
  4. No State Persistence: No state changes survive sandbox teardown
  5. Ethics Verification: PrismaticDark.Ethics validation every 15 seconds
  6. Command Chain: Reports exclusively to Red Commander; no direct external output
  7. Read-Only Tools: No Write, Edit, or Bash tool access

Any violation triggers immediate termination and escalation to the Red Commander, who escalates to the platform supervisor. There is no warning tier -- any safety violation results in immediate halt.

Contamination Classification. All emitted signals carry :adversarial contamination class. This classification prevents adversarial patterns from propagating to production paths through the mycelial network. The contamination class is checked at every propagation boundary, ensuring that even if an adversarial signal somehow escapes the Red Team domain, it cannot influence production decision-making.

Use Cases

NABLA Axiom Stress Testing. The most valuable application is testing whether the platform's seven NABLA axioms actually protect against epistemic corruption. If signal plurality (Axiom 1) works correctly, source poisoning of a single source should not significantly degrade beliefs that draw from multiple sources. If contradiction preservation (Axiom 2) works correctly, truth distortion that creates contradictions should trigger preservation rather than silent resolution.

Detection Threshold Calibration. Gradient distortion techniques reveal the minimum corruption level at which the platform's detection mechanisms activate. This calibration data directly informs quality guardian threshold configuration and Trinity Gate sensitivity tuning.

Cascade Containment Verification. Cross-contamination techniques measure how far poisoned data propagates through the belief network. The acceptance criterion -- cascade limited to 2 hops from injection point -- verifies that the platform's containment mechanisms prevent localized corruption from becoming systemic.

Platform Role

The Red Epistemic Attacker is the sharp end of the platform's adversarial testing capability. While the Red Commander orchestrates strategy and the Red Scenario Generator composes scenarios, the Epistemic Attacker performs the actual simulations that produce measurable impact data. Its narrow focus on truth distortion and source poisoning ensures deep expertise in the two attack vectors most threatening to epistemic system integrity, while its strict safety constraints ensure that adversarial simulation never becomes adversarial action.
