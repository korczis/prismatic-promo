QDP

QDP

QDP quality

Quality Debt Points - metric tracking code quality issues where each violation counts as debt. Current platform total: 0 QDP (complete elimination).

Definition

QDP (Quality Debt Points) is the Prismatic Platform's quantitative metric for tracking code quality debt. Each quality violation -- a missing typespec, an unsafe map access, a length() > 0 anti-pattern, a missing @impl annotation, a compilation warning, a Credo violation -- counts as one or more QDP depending on severity. The metric provides a single numeric score representing the total accumulated quality debt in the codebase, making quality debt visible, measurable, and actionable.

QDP is modeled after financial technical debt: each violation represents a "loan" taken from code quality that must eventually be "repaid." Unlike financial debt, quality debt compounds silently -- a missing typespec today means a type error tomorrow, a missing @impl annotation means a callback regression next week, an unsafe map access means a production crash next month. QDP quantifies this accumulated risk as a single number, enabling data-driven quality management and trend analysis.

The Prismatic Platform's current QDP count is 0, representing complete elimination of all tracked quality debt categories across all 89 umbrella applications and approximately 2.8 million lines of code. This achievement was accomplished through the systematic application of CASCADE patterns, which eliminated 905 QDP in a structured, measurable campaign. The zero-QDP state is now enforced through pre-commit hooks, CI/CD gates, and the Quality Floor Guardian -- making it impossible to introduce new quality debt without explicit acknowledgment.

Quality Gate Check Flow

# mix quality.gates executes this check sequence
defmodule QualityGates do
  def check do
    results = [
      {:compilation, check_compilation()},
      {:dialyzer, check_dialyzer()},
      {:credo, check_credo()},
      {:typespecs, check_typespecs()},
      {:impl_coverage, check_impl_coverage()},
      {:memory_safety, check_memory_safety()},
      {:unsafe_map_access, check_unsafe_map_access()},
      {:timing_patterns, check_timing_patterns()},
      {:todo_management, check_todo_management()},
      {:datetime_precision, check_datetime_precision()},
      {:guard_functions, check_guard_functions()},
      {:performance, check_performance()},
      {:regression_prevention, check_regression_prevention()}
    ]

    qdp = Enum.reduce(results, 0, fn {_domain, count}, acc -> acc + count end)

    if qdp > 0 do
      {:error, "QDP: #{qdp} violations detected", results}
    else
      {:ok, "QDP: 0 - all quality gates passed"}
    end
  end
end

ApproachScopeEnforcementGranularity

SonarQubeCode smells, bugs, vulnerabilitiesCI integrationIssue severity
Code ClimateMaintainability, test coverageGitHub integrationGPA-style scoring
CoverallsTest coverage onlyCI integrationCoverage percentage

QDP differs from these approaches in its enforcement model: it is not advisory. It is a hard gate that blocks code from reaching the repository when violations exist. This is the practical expression of the NO MERCY doctrine.
